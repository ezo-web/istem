# Security Policy

If you encounter a security vulnrability, contact the project lead (Ezo K) at your most imedeate convinience. The I-STEM Comms website is ment to be as safe and secure as possible.
A security vulnrability is something that meets this critera:
1. The issue has a possibility of exposing personal information (defined in wiki)
2. The issue is part of the HDSB I-STEM Comms website _system_:

    * The website's codebase
    * Services the website uses
    * Github configurations/settings
    * Any other service that communicate with the website
  
3. The issue compromises website/system security in any way not already listed
